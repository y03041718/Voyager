# 打印/导出PDF功能使用指南

## 功能概述

使用浏览器原生打印功能，用户可以将旅行计划打印或导出为PDF文件，完美保留页面样式，支持中文无乱码。

## 使用方法

### 方式一：点击按钮打印

1. 进入行程详情页面（Itinerary）
2. 点击右上角的"打印/导出PDF"按钮
3. 浏览器会打开打印对话框
4. 选择打印机或"另存为PDF"
5. 调整打印设置（建议勾选"背景图形"）
6. 点击"打印"或"保存"

### 方式二：使用快捷键

- Windows/Linux: `Ctrl + P`
- Mac: `Cmd + P`

## 打印设置建议

### Chrome/Edge浏览器
1. 目标打印机：选择"另存为PDF"（导出PDF）或实际打印机（直接打印）
2. 纸张尺寸：A4
3. 方向：纵向
4. 边距：默认
5. ✅ 勾选"背景图形"（重要！确保颜色和背景打印）
6. 缩放：100%

### Firefox浏览器
1. 打印机：选择"Microsoft Print to PDF"或"保存为PDF"
2. 纸张尺寸：A4
3. 方向：纵向
4. ✅ 勾选"打印背景"（重要！）

### Safari浏览器
1. 打印机：选择"存储为PDF"
2. 纸张尺寸：A4
3. 方向：纵向

## 打印内容

### 包含的内容
- ✅ 行程标题和基本信息（目的地、日期、出行人数、旅行风格）
- ✅ 每日行程详情（时间、地点、描述、评分、地址）
- ✅ 天气信息
- ✅ 当地特色与提示（文化、美食、实用建议）
- ✅ 景点图片
- ✅ 所有颜色和背景

### 自动隐藏的内容
- ❌ 导航栏和按钮
- ❌ 交互元素（分享、导出按钮等）
- ❌ 地图占位符（打印时不显示）

## 打印优化

### 自动优化
- 自动分页，避免内容被截断
- 每天的行程从新页开始（除第一天）
- 优化字体大小和间距适配纸张
- 移除阴影和动画效果
- 保留重要的颜色和边框

### 页面设置
- 纸张：A4 (210mm × 297mm)
- 方向：纵向
- 边距：上下15mm，左右10mm

## 常见问题

### Q: 为什么打印出来没有颜色？
A: 请在打印设置中勾选"背景图形"或"打印背景"选项。

### Q: 如何只打印某一天的行程？
A: 目前打印会包含所有天数的行程。如需单独打印某天，可以：
1. 使用浏览器的打印预览
2. 在打印对话框中选择特定页码

### Q: 打印出来的图片不清晰？
A: 建议：
1. 确保打印质量设置为"高"或"最佳"
2. 如果导出PDF，可以在PDF阅读器中调整打印质量

### Q: 可以横向打印吗？
A: 可以，在打印设置中将方向改为"横向"即可。但建议使用纵向以获得最佳效果。

### Q: 打印时能否显示页码？
A: 浏览器会自动在页脚添加页码。如需自定义，可以在打印设置的"更多设置"中调整页眉页脚。

### Q: 如何批量打印多个行程？
A: 需要分别打开每个行程页面进行打印。未来可能会添加批量导出功能。

## 技术实现

### 核心技术
- 浏览器原生打印API：`window.print()`
- CSS打印样式：`@media print`
- 无需第三方库，零依赖

### 优势
- ✅ 完美支持中文，无乱码
- ✅ 样式100%还原网页效果
- ✅ 文件更小，加载更快
- ✅ 用户可以自由选择打印机或导出PDF
- ✅ 支持所有现代浏览器

### 相关文件
- `src/print.css`：打印样式定义
- `src/pages/Itinerary.tsx`：行程页面（包含打印按钮）
- `src/main.tsx`：引入打印样式

## 开发者说明

### 添加不打印的元素
给元素添加 `no-print` 类：
```tsx
<button className="no-print">这个按钮不会被打印</button>
```

### 修改打印样式
编辑 `src/print.css` 文件：
```css
@media print {
  /* 自定义打印样式 */
  .my-element {
    font-size: 12pt;
  }
}
```

### 监听打印事件
```typescript
useEffect(() => {
  const beforePrint = () => {
    console.log('准备打印...');
  };
  
  const afterPrint = () => {
    console.log('打印完成');
  };
  
  window.addEventListener('beforeprint', beforePrint);
  window.addEventListener('afterprint', afterPrint);
  
  return () => {
    window.removeEventListener('beforeprint', beforePrint);
    window.removeEventListener('afterprint', afterPrint);
  };
}, []);
```
